    UPI Method — variables.tf¶

    All variable declarations for the UPI baremetal deployment (upi-method/).

    UPI-Specific Variables¶

    These variables are unique to UPI and do not exist in the IPI deployment:

    Variable Type Description
    install_dir string Directory on bastion for install artifacts
    rhcos_iso_url string HTTP URL to RHCOS live ISO
    rhcos_rootfs_url string HTTP URL to RHCOS rootfs image
    install_disk string Target disk for RHCOS installation
    boot_method string pxe, iso, or manual
    ignition_http_port number Port for ignition HTTP server
    bootstrap_ip string Bootstrap node IP address
    bootstrap_mac string Bootstrap node MAC address

    Node Definition Differences¶

    UPI nodes do not have BMC fields (bmc_address, bmc_username, bmc_password):

    master_nodes = [
  {
    name            = "master-0"
    ip              = "10.142.41.101"
    mac_address     = "aa:bb:cc:dd:ee:01"
    boot_mode       = "UEFI"
    bond_interfaces = ["ens1f0", "ens1f1"]
  },
]

    master_nodes = [
  {
    name             = "master-0"
    bmc_address      = "idrac-virtualmedia://10.142.41.100/redfish/v1/Systems/System.Embedded.1"
    bmc_username     = "admin"
    bmc_password     = "REPLACE_PASSWORD"
    boot_mac_address = "aa:bb:cc:dd:ee:01"
    boot_mode        = "UEFI"
    root_disk_min_gb = 890
    ip               = "10.142.41.101"
    bond_interfaces  = ["ens1f0", "ens1f1"]
  },
]
